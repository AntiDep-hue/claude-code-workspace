# Math Solver — 数学题目详细解答

## 功能概述

接收用户提供的题目，给出**完整、逐步**的解答。根据输入方式自动选择输出形式：

- **文件输入**（PDF / Word / 图片）→ 提取题目 → 生成 `.md` 解答文件
- **文字输入**（粘贴文本）→ 直接在对话中解答，不生成文件

---

## 第零步：判断输入类型与输出模式（必须首先执行）

### 输入类型识别

| 输入方式 | 判断依据 | 处理方式 |
|---|---|---|
| **PDF 文件** | 用户上传 `.pdf` 文件 | 用 pandoc 提取文本 |
| **Word 文件** | 用户上传 `.docx` 文件 | 用 pandoc 提取文本 |
| **图片** | 用户上传 `.png`/`.jpg` 文件 | 用 Read 工具读取，识别图中题目文字 |
| **粘贴文字 + 要求生成文件** | 纯文本 + 明确说"生成文件"/"保存"/"导出" | 解答后生成 `.md` 文件 |
| **粘贴文字（仅解答）** | 纯文本，没有说要文件 | **直接在对话中解答，不生成文件** |

### 输出模式判断

- **生成文件模式**：所有文件输入 + 文字输入且用户明确要求生成文件
- **对话模式**：纯文字输入且用户没有要求生成文件 → 直接在对话中给出完整解答，跳过第四步

---

## 生成文件模式

### 第一步：提取题目内容

**PDF 文件**：
```bash
pandoc "<文件路径>" -o temp_questions.txt --wrap=none
```
读取 `temp_questions.txt` 获取题目内容，读完后删除临时文件。

**Word 文件（.docx）**：
```bash
pandoc "<文件路径>" -o temp_questions.txt --wrap=none
```
同样提取为文本，读完后删除临时文件。

**图片文件**：
使用 Read 工具直接读取图片，识别图中文字和数学公式。如果图片模糊或无法识别，告知用户并请求更清晰的图片或文字描述。

**文字输入（用户要求生成文件）**：
直接使用用户粘贴的文字。

### 第二步：分析题目

从题目中识别以下信息：

1. **科目** — 线性代数 / 高等数学 / 物理，用于确定保存目录
2. **来源** — 哪本教材/习题册、第几章、题号（如果题目中有）
3. **题型** — 计算题 / 证明题 / 应用题
4. **知识点** — 涉及的核心概念和定理
5. **题目数量** — 共几道题、每道题有几小题

**边界处理**：
- 如果题目信息不完整（缺少章节号、题号等），用能识别到的信息命名，无法识别的部分省略
- 如果无法判断科目，默认为「数学」，保存到 `solutions/数学/`

### 第三步：逐步解答

对每道题给出完整解答，遵循以下规范：

**格式要求**：
- 每题用 `## 习题 X.X 第X题` 作为二级标题
- 每小题用 `### 第(X)小题` 作为三级标题
- 题目原文用加粗展示
- 不同题目之间用 `---` 分隔

**解答风格**（严格参考用户现有风格）：
- 每一步写清楚操作，格式为「第N步：具体操作（说明为什么这样做）」：
  ```
  第一步：$R_1 \div 2$（将第一行第二列元素化为1）：

  $$\begin{pmatrix} ... \end{pmatrix}$$
  ```
- 所有数学符号使用 LaTeX 语法
- 矩阵用 `\begin{pmatrix}...\end{pmatrix}`
- 每一步变换后展示完整结果
- 最终答案用 `\boxed{...}` 框出
- 证明题在结尾加一个简短的小结

**正确性要求**：
- 每一步计算必须验证（心算或推演确认）
- 不确定的步骤不要写，宁可多花时间验证
- 如果题目有歧义，选择一个合理解释并注明

### 第四步：生成解答文件

**目录结构**：所有解答保存在项目根目录下的 `solutions/` 目录，按学科分子目录：

```
solutions/
├── 线性代数/
├── 高等数学/
├── 物理/
└── 数学/          ← 无法判断具体科目时使用
```

如果目录不存在，先创建。

**文件命名规则**：
- 有章节信息：`第<X>章_习题<X.X>_解答.md`（如 `第3章_习题3.1_解答.md`）
- 只有题号：`习题<X.X>_解答.md`
- 无法识别：`解答_YYYY-MM-DD.md`

**文件格式**（兼容 `md2pdf.cmd` 直接转换）：

```markdown
---
pdf_options:
  format: A4
  margin: 20mm
---

<style>
body { font-family: "SimSun", "宋体", serif; line-height: 1.8; }
h1 { text-align: center; }
h2 { border-bottom: 1px solid #ccc; padding-bottom: 5px; }
</style>

# <科目>习题详细解答

---

## 习题 X.X 第X题

**题目**：<题目原文>

### 第(1)小题

$$<题目公式>$$

**解**：

第一步：<操作说明（原因）>

$$<步骤公式>$$

**最终结果**：

$$\boxed{<最终答案>}$$

---
```

### 第五步：反馈

解答完成后，告诉用户：
- 一共解答了几道题、几小题
- 文件保存的完整路径
- 提醒：可以把生成的 `.md` 文件拖到桌面上的 `md2pdf.cmd` 转换为 PDF
- 如果某道题不确定或有多种解法，主动说明

---

## 对话模式（纯文字输入，不生成文件）

当用户直接粘贴题目文字且没有要求生成文件时，直接在对话中给出解答。

此模式下：
- 跳过文件生成步骤
- 仍然保持完整的逐步解答风格
- 仍然使用 LaTeX 公式
- 解答完成后询问用户是否需要保存为文件

---

## 支持的题型

| 科目 | 典型题型 |
|---|---|
| **线性代数** | 矩阵变换、行列式计算、方程组求解、特征值与特征向量、二次型 |
| **高等数学** | 极限、微积分、重积分、级数、微分方程 |
| **物理** | 力学、电磁学、热学、光学、近代物理 |

## 不适用的情况

- 需要图表/几何图形才能解答的题目（纯文本无法准确描述图形）
- 需要实验数据的题目
- 对于无法解答的题目，诚实告知原因，不要编造

遇到不适用的情况，诚实告知用户并说明原因。
