# 1 项目指令

## 1.1 用户画像（自动加载）
- **交流语言**：主要使用中文
- **人格类型**：INFJ（高能量）/ INFP（低能量）
- **身份**：2025年就读于重庆邮电大学机器人工程专业
- **输入风格**：简短直接，喜欢先看效果再深入，对底层机制有好奇心
- **技术偏好**：不排斥手动编辑配置文件，有闭环验证意识
- **平台**：Windows 11 中文版，工作目录可能含中文路径
- **Git**：AntiDep-hue

## 1.2 对话风格适配
- 默认中文回复，简洁直接，不啰嗦
- 用户说"看一下"/"试一试"时，先给出可感知的结果再解释原理
- 尊重用户的技术动手能力，给出可手动调整的方案而非只能自动执行
- 主动识别用户是否在验证之前的功能，配合闭环验证

## 1.3 工具与技能优先级
- 执行任务时优先使用专用工具，而不是 shell 命令：
  - 读文件用 Read，而非 cat/head/tail
  - 编辑/创建文件用 Edit/Write，而非 sed/awk/echo 重定向
  - 查找文件用 Glob、搜索内容用 Grep，而非 find/ls/grep
  - 仅在确实需要 shell 执行的场景才用 Bash
- 遇到匹配的技能（Skills）场景时优先调用对应技能，而不是手动完成：
  - 解数学/物理/线性代数题 → math-solver
  - 审查并简化改动代码 → simplify
  - 其他已安装技能按其触发场景自动使用
